Fix Metadata

Scans every track in your Rekordbox database and compares two cached values against the actual file on disk: FileType (the format code — MP3, FLAC, WAV, etc.) and FileSize (the byte count). If either value is wrong, it is corrected in the database. File paths and audio content are never modified.

⚠ Close Rekordbox before running. A timestamped backup of master.db is created automatically before any changes are written.

What this corrects:

FileType The format code Rekordbox uses to identify the file (e.g. 1 = MP3, 6 = FLAC). Becomes wrong when a file is converted to a different format but the database is not updated.
FileSize The byte count cached in the database. Used by Rekordbox to detect file changes. Becomes wrong after re-encoding, replacing a file in place, or any operation that changes the file's size on disk.

Use this after running Relocate Tracks, or any time Rekordbox shows tracks as broken despite the files being present on disk.
⚑ Re-analysis required after running this tool.

Fixing FileType and FileSize in the database corrects how Rekordbox identifies your files, but Rekordbox does not automatically re-read waveform, BPM, or key data for updated tracks. After running this tool you should:

1. Open Rekordbox
2. Select all affected tracks (you can use the report below to identify them)
3. Right-click → Analyze Tracks

Re-analysis time depends on library size — expect roughly 1–3 minutes per 100 tracks on a modern machine.
