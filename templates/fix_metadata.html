Fix Metadata

Correct stale FileType and FileSize in the Rekordbox database for tracks whose file path is correct but whose cached metadata doesn't match the actual file on disk.

⚠ Close Rekordbox before running. A timestamped backup of master.db is created automatically before any changes are written.

This tool does not move or modify any audio files — it only corrects the FileType and FileSize values stored in the database. Use this after running Relocate Tracks, or any time Rekordbox shows tracks as broken despite the files being present on disk.
⚑ Re-analysis required after running this tool.

Fixing FileType and FileSize in the database corrects how Rekordbox identifies your files, but Rekordbox does not automatically re-read waveform, BPM, or key data for updated tracks. After running this tool you should:

1. Open Rekordbox
2. Select all affected tracks (you can use the report below to identify them)
3. Right-click → Analyze Tracks

Re-analysis time depends on library size — expect roughly 1–3 minutes per 100 tracks on a modern machine.

How to restore a Rekordbox database backup

Every tool that modifies the database creates a timestamped backup before writing anything. Here's how to restore if something went wrong.

  1. Close Rekordbox completely before touching any database files.
  2. Open File Explorer and paste this into the address bar:
    %APPDATA%\Pioneer\rekordbox\
  3. You'll see master.db (live) and timestamped backups like master_backup_20250415_143022.db.
  4. Rename master.db → master_broken.db to keep it as a reference.
  5. Rename the backup you want to restore → master.db.
  6. Open Rekordbox — it loads from the restored file.

⚑ Delete old backups periodically to save disk space.
