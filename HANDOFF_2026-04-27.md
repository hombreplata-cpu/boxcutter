### 2. Add a file-trace diagnostic to launcher.py

Replace the existing `print(...)` diagnostic with a write to a deterministic file location that nothing else can write to:

```python
# Place at the VERY TOP of launcher.py — before any imports of app, webview,
# or anything else from the project. After only stdlib imports.
from pathlib import Path as _DiagPath

_diag_path = _DiagPath("/tmp/boxcutter-launcher-trace.txt") if sys.platform != "win32" \
             else _DiagPath(os.environ.get("TEMP", "C:\\Temp")) / "boxcutter-launcher-trace.txt"
try:
    _diag_path.write_text(
        f"launcher started\n"
        f"BOXCUTTER_TESTING={os.environ.get('BOXCUTTER_TESTING', '<unset>')!r}\n"
        f"BOXCUTTER_PORT={os.environ.get('BOXCUTTER_PORT', '<unset>')!r}\n"
        f"frozen={getattr(sys, 'frozen', False)}\n"
        f"argv={sys.argv}\n"
        f"sys.executable={sys.executable}\n"
    )
except Exception:
    pass
```

Why a file: bypasses any stdout/stderr quirks PyInstaller has on either OS. If the file appears, the launcher executed. If the file content shows the env vars correctly, the bundle is reading them. If the file appears but env shows `<unset>`, env propagation is broken (probe shell vs subprocess). If the file does NOT appear, either the launcher.py shipped is different from what I committed, OR the launcher never executes my code.
